Jalon 3 — Frigate NVR avec GPU Passthrough Validé
VM Debian 12 avec GTX 1050 Ti en PCI passthrough (IOMMU activé, groupe IOMMU isolé, driver NVIDIA 555.261.03, CUDA 12.2). Frigate déployé via Docker Compose, GPU visible dans le conteneur (nvidia-smi OK). HDD 1 To monté sur /media/frigate. // Résolution de Problèmes — Méthode STAR
// Situation

Infrastructure multi-VMs sur serveur physique avec Déploiement finalisé sur 8 Go suite à une défaillance matérielle, mais une extension à 64 Go prévue prochainement. GPU NVIDIA dédié à l'analyse vidéo IA. Les workloads (firewall, domotique, NVR, administration) et les équipements IoT devaient coexister sur le même hyperviseur sans faille d'isolation réseau.

// Tâche

Concevoir une architecture segmentée de type professionnel en contrainte matérielle forte : GPU en passthrough exclusif, 4 VLANs étanches avec politiques firewall distinctes, et plan d'extension Wi-Fi pour couvrir l'intégralité du réseau domestique.

// Action

Déploiement par jalons documentés, avec validation avant chaque étape. Incidents réels traités : EFI Secure Boot bloquant (résolu), 401 RTSP caméra (résolu), snapshot Proxmox impossible en GPU passthrough (contournement documenté). Validation ONVIF/PTZ en local pour éliminer la dépendance cloud des caméras.

// Résultat

4 VMs stables en production, GPU passthrough fonctionnel, caméra RTSP sans accès cloud, 4 VLANs isolés avec règles firewall opérationnelles. Infrastructure documentée, évolutive, et prête pour l'extension réseau switch + AP Wi-Fi.
